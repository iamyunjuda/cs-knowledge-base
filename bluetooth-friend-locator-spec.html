1. 프로젝트 개요

항목내용
앱 이름 (가칭)Nearby
플랫폼iOS 17.0+
언어Swift 5.9+

2. 핵심 사용자 시나리오

시나리오 흐름

① 앱 설치 후 친구 1명과 페어링 (초대 코드 / QR)
② 앱을 백그라운드에 두고 일상 생활
③ 친구가 BLE 범위(~50m)에 진입 → 푸시 알림: "친구가 근처에 있어요!"
④ 홈 화면 위젯 / Live Activity에서 나침반 방향 + 거리 실시간 확인
⑤ 친구가 범위를 벗어나면 → 위젯: "범위 밖" 표시, 다시 대기 모드

3. 기능 요구사항 (MoSCoW)

MUST 필수   SHOULD 권장   COULD 선택

3.1 인증 / 친구 관리

ID우선순위기능상세
F-01MUST회원가입 / 로그인Apple Sign In 지원. 최소한의 프로필 (닉네임, 프로필 이미지)

3.2 BLE 근접 감지

ID우선순위기능상세

F-14SHOULD거리 구간 표시"바로 옆" / "가까움" / "주변" / "멀어지는 중" 텍스트 + 색상 변경

4. 기술 아키텍처

── iOS Client ──
Swift 5.9+ / SwiftUI / iOS 17.0+
Core Bluetooth  → BLE Advertising & Scanning
Core Location   → iBeacon Monitoring, GPS, Heading
WidgetKit       → 홈 화면 위젯
ActivityKit     → Live Activity + Dynamic Island
Nearby Interaction → UWB (선택적)

── Backend ──
Firebase Auth    → Apple Sign In
Firestore       → 유저/친구 관리
WebSocket (서버)  → 실시간 GPS 좌표 교환
FCM             → 원격 푸시 알림 (보조)

── 데이터 흐름 ──
App Group (UserDefaults) → 메인 앱 ↔ Widget Extension
ActivityKit push        → 서버 → Live Activity 업데이트

6. 상태 전이 다이어그램

상태진입 조건동작전이
Idle앱 시작 / 친구 이탈iBeacon Region Monitoring만 활성. 저전력→ Detected (BLE 감지 시)
Detected친구 BLE 신호 감지Local Notification 발송. GPS 활성화. WebSocket 연결→ Tracking (좌표 수신 시)
Tracking양쪽 GPS 좌표 교환 시작RSSI 거리 측정 + 방위각 계산. 위젯/Live Activity 업데이트 (1~3초)→ Lost (신호 소실 시)
LostBLE 신호 30초간 미수신"범위 밖" 위젯 표시. GPS/WebSocket 중지. 알림 선택적 발송→ Idle (60초 후)

9. 비기능 요구사항

항목요구사항
배터리백그라운드 대기 시 하루 5% 이하 추가 소모. GPS는 Tracking 상태에서만 활성화
반응 속도BLE 감지 후 알림까지 10초 이내. 나침반 업데이트 1~3초 주기
정확도거리: ±5m 이내 (실내 기준). 방향: ±15° 이내 (GPS 정확도 의존)
보안UUID는 서버에서 발급. GPS 좌표는 암호화 전송. 좌표 서버 미저장 (실시간 릴레이만)
최소 사양iOS 17.0+, iPhone SE 2세대 이상

10. 리스크 및 대응

리스크영향대응
iOS 백그라운드 BLE 제한감지 지연 (수 분)iBeacon Region Monitoring 사용. 서버 Push 알림 보조
RSSI 거리 부정확사용자 혼란칼만 필터 + "약 ~m" 표현. 구간 텍스트 병행 표시
GPS 실내 오차방향 부정확실내 감지 시 나침반 숨기고 거리만 표시. UWB 폴백
배터리 소모사용자 이탈상태별 센서 On/Off 전략. Instruments로 프로파일링
Apple 심사 리젝출시 지연Always Location 사용 이유 명확히 기술. 스크린샷 준비
